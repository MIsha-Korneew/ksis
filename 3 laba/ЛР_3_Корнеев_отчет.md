## 3. Краткие теоретические сведения

### 3.1. Порт

**Порт** — числовой идентификатор (0…65535), который позволяет ОС **на одном IP‑адресе** различать **несколько сетевых приложений**. Для клиента при исходящем TCP/UDP соединении ОС назначает **временный (эфемерный) порт**; сервер обычно слушает **фиксированный** порт (в работе: TCP **5000**, UDP **5001**).

### 3.2. TCP

**TCP** — протокол с **установлением соединения**, **надёжной** доставкой и упорядочиванием байтов. Установление соединения: **трёхстороннее рукопожатие** (SYN → SYN‑ACK → ACK). Подходит для **чата**: поток сообщений между клиентом и сервером.

### 3.3. UDP

**UDP** — протокол **без соединения**, передаёт **датаграммы**. Доставка не гарантирована, порядок не гарантирован. Подходит для **коротких служебных сообщений** (в работе — **поиск сервера** и **широковещательные объявления**).

### 3.4. Broadcast

**Широковещательная (broadcast) доставка UDP** — отправка на адрес вида **255.255.255.255** (или broadcast подсети): датаграмма распространяется **в пределах L2/L3 сегмента**, как правило **не маршрутизируется** в Интернет. Используется для **обнаружения сервиса** в локальной сети.

### 3.5. Клиент–серверная модель

**Сервер** принимает подключения и обслуживает **многих клиентов**. **Клиент** инициирует подключение к известному **IP и порту**. В данной работе сервер **ретранслирует** сообщения всем подключённым клиентам.

### 3.6. Многопоточность

Для одновременной работы с несколькими клиентами на сервере для **каждого** TCP‑подключения создаётся **отдельный поток**. На клиенте отдельный поток используется для **постоянного приёма** сообщений, пока в основном потоке выполняется ввод с клавиатуры.

## 4. Описание программной реализации

Разработаны два модуля: **`chat_server.py`** (сервер) и **`chat_client.py`** (клиент). Используется стандартный модуль Python **`socket`**, **`threading`**.

### 4.1. Сервер (`chat_server.py`)

- Создаётся **TCP‑сокет** `SOCK_STREAM`, привязка к `host:port` (по умолчанию `0.0.0.0:5000`), `listen`.
- Для каждого принятого соединения `accept()` запускается **поток** `handle_client`.
- Общий список клиентов и рассылка сообщений защищены **`threading.Lock`**.
- Параллельно поднимается **UDP‑сокет** на порту **`--udp-port`** (по умолчанию **5001**):
  - при получении **`KSIS_DISCOVER`** отвечает **unicast** строкой **`KSIS_TCP|<IP>|<TCP_PORT>`**;
  - периодически (по умолчанию каждые 5 с) шлёт **broadcast** **`KSIS_ANN|<IP>|<TCP_PORT>`** на `255.255.255.255`.
- IP для объявлений определяется функцией **`detect_local_ip()`** (через вспомогательный UDP `connect` к `8.8.8.8` и `getsockname()`), либо задаётся вручную **`--announce-ip`**.

### 4.2. Клиент (`chat_client.py`)

- Режим **`--discover`**: отправка **`KSIS_DISCOVER`** UDP broadcast на порт **5001**, ожидание ответа **`KSIS_TCP|...`** или перехват **`KSIS_ANN|...`**, затем **TCP `connect`** к полученному адресу.
- Обычный режим: **TCP `connect`** к указанным **host** и **port**.
- После соединения отправляется первый кадр **`JOIN <ник>`**.
- Запускается поток **`reader`** для приёма кадров и вывода в консоль.
- Ввод пользователя: строки отправляются как **`MSG <текст>`**; выход — **`/quit`** → кадр **`QUIT`**.

### 4.3. Формат сообщений (прикладной протокол поверх TCP)

Каждое сообщение — **4 байта длины** (unsigned int, **big‑endian**) + **полезная нагрузка** в **UTF‑8**.

Текстовые команды:

| Направление | Примеры |
|-------------|---------|
| Клиент → сервер | `JOIN <nick>`, `MSG <text>`, `QUIT` |
| Сервер → клиенты | `SYS <текст>`, `MSG <nick>: <текст>` |

### 4.4. Формат UDP (discovery / announce)

Строки в **UTF‑8**, завершающий перевод строки `\n`:

| Сообщение | Назначение |
|-----------|------------|
| `KSIS_DISCOVER` | запрос «где сервер?» (broadcast) |
| `KSIS_TCP\|<IP>\|<port>` | ответ сервера клиенту (unicast) |
| `KSIS_ANN\|<IP>\|<port>` | периодическое объявление сервера (broadcast) |

## 5. Средства разработки и запуска

- ОС: **Windows 10/11**  
- Язык: **Python 3**  
- Запуск из каталога с файлами, например:  
  `C:\Users\User\ksis_clone\3 laba\`

**Сервер:**

```text
cd /d C:\Users\User\ksis_clone\3 laba
py chat_server.py 0.0.0.0 5000
```

**Клиент (ручной IP):**

```text
py chat_client.py 127.0.0.1 5000 --nick <Имя>
```

**Клиент (поиск по UDP):**

```text
py chat_client.py --discover --nick <Имя>
```

Выход из чата: **`/quit`**
