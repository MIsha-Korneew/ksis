# ЛР4 — HTTP-прокси

## Файлы

- `http_proxy.py` — прокси на сокетах, потоки на клиента; журнал в консоли: `URL -> код`.

---

## Как правильно запускать

### 1. Терминал

1. Открой **cmd** или PowerShell.
2. Перейди в папку с программой (путь подставь свой):

```text
cd /d D:\бугор\ksis\4 laba
```

3. Запусти прокси (порт можно сменить, если `8080` занят):

```text
py http_proxy.py --port 8080
```

или `python http_proxy.py --port 8080`, если команда `py` не настроена.

4. В консоли должно появиться что-то вроде: `HTTP proxy слушает 0.0.0.0:8080`. **Это окно не закрывай** — пока оно работает, прокси включён.
5. Остановка: **Ctrl+C** в этом окне.

### 2. Порт

- По умолчанию слушается **`8080`**. Если пишет «Address already in use» / «уже используется», либо закрой программу, которая заняла порт, либо запусти так:

```text
py http_proxy.py --port 8888
```

и в настройках браузера укажи **тот же** порт (`8888`).

### 3. Права и файрвол

- Обычно **достаточно прав обычного пользователя**. Если Windows спросит доступ в сети — разреши **частные сети** для `python`.
- Антивирус/корпоративный файрвол иногда блокирует локальный прослушивание — тогда добавь исключение или временно проверь с отключённым файрволом (только для отладки).

### 4. Браузер: только HTTP-прокси

Прокси умеет **только HTTP** (не HTTPS-CONNECT). Поэтому:

- В настройках Windows / браузера укажи **ручной прокси**:
  - **Адрес:** `127.0.0.1`
  - **Порт:** тот же, что в консоли (например `8080`)
- Лучше **не** включать «использовать этот прокси для **HTTPS**» или **отдельно** не задавать HTTPS на тот же порт: сайты с `https://` пойдут в обход или дадут ошибку — **для проверки лабы** открывай ссылки **именно с `http://`** (см. ниже).
- **Chrome / Edge (Windows):**  
  `Параметры` → `Система` → `Открыть настройки прокси` → `Настройка вручную` → `Использовать прокси-сервер` → `127.0.0.1` и порт. Либо классическое окно: **Настройки сети Internet** → **Подключения** → **Настройка LAN** → **Прокси-сервер**.

### 5. Какие ссылки открывать (из задания)

Проверка работоспособности — **только HTTP:**

- `http://example.com/`
- `http://live.legendy.by:8000/legendyfm` (длинный поток — оставь вкладку открытой, не прерывай сразу)

### 6. Что должно быть в консоли

После открытия страницы в браузере в окне прокси появляются строки вида:

```text
http://example.com/ -> 200
```

При ошибке соединения с сервером:

```text
http://... -> ERR (...)
```

### 7. Типичные проблемы

| Проблема | Что сделать |
|----------|-------------|
| «Не удаётся получить доступ к сайту» | Прокси не запущен, неверный IP/порт в браузере, или в адресе `https://` вместо `http://` для теста. |
| Сайт не открывается через прокси | Убедись, что **сначала** запущен `http_proxy.py`, **потом** открывается вкладка. |
| Порт занят | Запусти с другим `--port` и поменяй порт в настройках Windows/браузера. |
| 501 / HTTPS | Браузер пытается идти на HTTPS через прокси. Для лабы тестируй **http://**-URL. |

### 8. Wireshark (для отчёта/защиты)

- Захват на **Adapter for loopback** (для `127.0.0.1`) или на своём **Wi-Fi**, если смотришь реальный IP.
- Фильтр, например: `tcp.port == 8080` (трафик браузер → прокси) и далее внешние `tcp.port == 80` / `8000` к серверу назначения.

## Суть протокола (кратко)

- Браузер шлёт на прокси: `GET http://хост/путь HTTP/1.1` (полный URL).
- Прокси подключается к серверу назначения и передаёт: `GET /путь` и заголовок `Host:` (см. [RFC 2616 §5.1.2](https://tools.ietf.org/html/rfc2616#section-5.1.2)).
