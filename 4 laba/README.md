# ЛР4 — HTTP-прокси

## Файлы

- `http_proxy.py` — прокси на сокетах, потоки на клиента; журнал `URL -> код`.

## Запуск

```text
py http_proxy.py --port 8080
```

## Браузер

- Укажи **только HTTP-прокси** `127.0.0.1:8080` (в Chrome: «Прокси-сервер настройки вручную»; **HTTPS** для тестов задания не обязателен и может мешать).
- Открой: `http://example.com/`, `http://live.legendy.by:8000/legendyfm` (радио — проверка длинного потока).

## Суть протокола

- Браузер шлёт на прокси запросы вида `GET http://хост/путь HTTP/1.1` (абсолютный URL, [RFC 2616 §5.1.2](https://tools.ietf.org/html/rfc2616#section-5.1.2)).
- Прокси подключается к серверу назначения и передаёт `GET /путь HTTP/1.1` и заголовок `Host:` с нужным хостом/портом.

## Wireshark (защита)

- Фильтр, например: `ip.addr == 127.0.0.1` или твой IP + `tcp.port == 8080` к прокси и исходящий HTTP на 80/8000.
- Видно: клиент (браузер) ↔ прокси ↔ origin.
